    1.3 Visión general de las vulnerabilidades web más comunes (OWASP Top 10)¶

    ¿Qué es el OWASP Top 10?¶

    OWASP (Open Web Application Security Project) es una organización que actualiza regularmente una lista de los 10 riesgos más críticos en aplicaciones web. La versión actual es de 2021, pero tienen previsto actualizarla este verano u otoño.

    OWASP Top 10

    En la imagen anterior se pueden ver las diferencias entre las versiones de 2017 y 2021.

    1. Controles de acceso rotos (Broken Access Control)¶

    Ocurre cuando falla la restricción de que los usuarios solo puedan realizar acciones para las que están autorizados. OWASP Top 10 - Controles de Acceso Rotos

    Ejemplos¶

    • Acceder a archivos privados sin autorización
    • Editar datos de otros usuarios
    • Tener acceso a funciones administrativas sin los permisos adecuados

    Prevención¶

    • Implementar controles de acceso de forma prioritaria
    • Comenzar con una política de "denegar por defecto"
    • Verificar los permisos en el lado del servidor

    Ejercicios¶

    • Broken Access Control

    2. Fallos criptográficos (Cryptographic Failures)¶

    Vulnerabilidades o errores en la implementación de la criptografía utilizada para proteger los datos. OWASP Top 10 - Fallos Criptográficos

    Ejemplos¶

    • Transmitir datos sin cifrar
    • Usar algoritmos de cifrado débiles (MD5, SHA1, DES...)
    • Mala gestión de claves privadas

    Prevención¶

    • Utilizar estándares de cifrado actuales (AES, RSA...)
    • Usar HTTPS siempre
    • Almacenar las claves de forma segura

    Ejercicios¶

    • Cryptographic Failures

    3. Inyección (Injection)¶

    La inyección es uno de los riesgos más graves para la seguridad de las aplicaciones web actuales. Los atacantes insertan datos maliciosos en comandos o consultas que la aplicación interpreta. De esta manera, pueden tomar el control de la aplicación, obtener datos personales o manipular la base de datos. OWASP Top 10 - Inyección

    Ejemplos¶

    • Inyección SQL (SQL Injection):

    Este es el tipo de ataque de inyección más común. Afecta a las bases de datos.

    -- Por ejemplo: Entrada del usuario: ' OR '1'='1' --

-- Consulta SQL resultante:

SELECT * FROM users WHERE username = '' OR '1'='1' -- AND password = '...'

-- El resto de la consulta después de nuestro código inyectado no se ejecutará.
    • Inyección de Comandos del Sistema Operativo (OS Command Injection):

    La inyección de comandos del sistema operativo no ocurre en el navegador, sino en el servidor. Para realizar el ataque, el atacante consigue insertar y ejecutar comandos en el sistema operativo del servidor a través del sitio web.

    ```bash -- Por ejemplo: ping 127.0.0.1 ; ls -l

    -- Imagina que una aplicación web tiene un formulario donde el usuario ingresa una dirección IP. La aplicación, en segundo plano, ejecuta el comando ping del sistema operativo para verificar si la dirección está disponible. Con el ejemplo proporcionado, después de hacer ping, al ejecutar ls -l, en sistemas Unix y Linux, este comando lista los archivos y carpetas del directorio actual.

    -- Consulta original: SELECT * FROM users WHERE username = 'admin' AND password = ''

    -- Consulta inyectada: SELECT * FROM users WHERE username = 'admin' --' AND password = '' ```

    En este caso, -- es un comentario en SQL, y el resto de la consulta no se ejecutará. Así, devolverá los datos del usuario con nombre de usuario "admin" sin verificar la contraseña.

    • Inyección de comandos (Command Injection):

    bash ; rm -rf / # Intenta ejecutar un comando malicioso

    • Inyección LDAP (LDAP Injection):

    ```bash -- Por ejemplo: )(uid=))(|(uid=*

    -- Imagina una aplicación web con un formulario de búsqueda que usa LDAP para autenticación. -- Un atacante podría inyectar esta entrada para modificar la consulta LDAP y obtener acceso no autorizado. -- En este ejemplo, la inyección intenta hacer coincidir cualquier usuario en el directorio LDAP. ```

    ### Prevención - Usar consultas parametrizadas (prepared statements) - Emplear ORMs seguros para evitar consultas SQL directas - Validar y desinfectar todas las entradas del usuario - Implementar el principio de mínimo privilegio en cuentas de base de datos - Usar procedimientos almacenados cuando sea posible - Validar y limpiar todas las entradas de usuario - Aplicar el principio de privilegio mínimo

    Ejercicios¶

    • Injection

    4. Diseño inseguro (Insecure Design)¶

    Problemas de seguridad que se originan en la fase de diseño, no solo en la implementación. OWASP Top 10 - Diseño Inseguro

    Ejemplos¶

    • Diseño sin principios de seguridad
    • Falta de modelado de amenazas
    • Ausencia de mecanismos contra el uso abusivo

    Prevención¶

    • Incorporar la seguridad desde la fase de diseño
    • Realizar modelado de amenazas
    • Identificar y prevenir casos de abuso

    Ejercicios¶

    • Insecure Design

    5. Configuración insegura (Security Misconfiguration)¶

    Falta de configuración de seguridad adecuada o configuraciones incorrectas en la aplicación. OWASP Top 10 - Configuración Insegura

    Ejemplos¶

    • Dejar configuraciones predeterminadas sin modificar
    • Mostrar mensajes de error detallados
    • Dejar desactivadas medidas de seguridad adicionales

    Prevención¶

    • Establecer una configuración segura
    • Automatizar el proceso de configuración
    • Actualizar y revisar la configuración regularmente

    Ejercicios¶

    • Security Misconfiguration

    6. Componentes vulnerables y desactualizados (Vulnerable and Outdated Components)¶

    Vulnerabilidades presentes en bibliotecas y componentes de terceros utilizados en la aplicación. OWASP Top 10 - Componentes Vulnerables y Desactualizados

    Ejemplos¶

    • No estar al tanto de vulnerabilidades conocidas
    • No aplicar actualizaciones disponibles
    • Utilizar componentes con fallos de seguridad conocidos

    Prevención¶

    • Mantener todos los componentes actualizados
    • Eliminar dependencias no utilizadas
    • Escanear regularmente en busca de vulnerabilidades

    Ejercicios¶

    • Vulnerable Components

    7. Fallos de identificación y autenticación (Identification and Authentication Failures)¶

    Problemas en la verificación de la identidad de los usuarios y en el proceso de autenticación. OWASP Top 10 - Fallos de Identificación y Autenticación

    Ejemplos¶

    • Aceptar contraseñas débiles
    • Mala gestión de sesiones
    • Funcionalidades de autenticación débiles

    Prevención¶

    • Implementar autenticación de múltiples factores (MFA)
    • Aplicar políticas de contraseñas robustas
    • Validar adecuadamente los identificadores de sesión

    Ejercicios¶

    • Identification and Authentication Failures

    8. Fallos en la Integridad del Software y Datos (Software and Data Integrity Failures)¶

    Problemas que comprometen la integridad de los datos y la confiabilidad del software. OWASP Top 10 - Fallos en la Integridad del Software y Datos

    Ejemplos¶

    • Modificación no autorizada de código o datos
    • Falta de validación de entradas de usuario
    • Código sin firmar digitalmente

    Prevención¶

    • Implementar firmas digitales
    • Verificar el origen del código y los datos
    • Validar la integridad de los datos

    Ejercicios¶

    • Software and Data Integrity Failures

    9. Fallos en el Registro y Monitoreo de Seguridad (Security Logging and Monitoring Failures)¶

    Falta de capacidad para detectar y responder a incidentes de seguridad. OWASP Top 10 - Fallos en el Registro y Monitoreo de Seguridad

    Ejemplos¶

    • Falta de configuración de registro de eventos
    • No recibir alertas de seguridad
    • No contar con procedimientos de respuesta a incidentes

    Prevención¶

    • Implementar un sistema de registro adecuado
    • Configurar alertas automáticas
    • Establecer procedimientos de respuesta a incidentes

    Ejercicios¶

    • Security Logging and Monitoring Failures

    10. Falsificación de Petición del Lado del Servidor (Server-Side Request Forgery - SSRF)¶

    Cuando un atacante puede controlar las peticiones que el servidor realiza a su red interna. OWASP Top 10 - Falsificación de Petición del Lado del Servidor

    Ejemplos¶

    • Realizar peticiones a servicios internos
    • Obtener información de configuración
    • Acceder a recursos de la red interna

    Prevención¶

    • Validar y filtrar todas las URLs de entrada
    • Utilizar listas blancas de dominios permitidos
    • Establecer límites de conexión y tiempo de espera

    Ejercicios¶

    • Server-Side Request Forgery
