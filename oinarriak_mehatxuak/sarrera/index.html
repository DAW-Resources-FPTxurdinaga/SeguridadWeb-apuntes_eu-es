    1.1 Introducción a la Ciberseguridad en el Desarrollo Web¶

    ¿Por qué es importante la ciberseguridad en el desarrollo web?¶

    Las aplicaciones web actualmente funcionan como repositorios de datos sensibles tanto para empresas como para usuarios. Implementar medidas de ciberseguridad ayuda a proteger:

    • La confidencialidad de los datos personales de los usuarios
    • La integridad y disponibilidad del sitio web
    • La reputación y confianza de la empresa
    • El cumplimiento normativo (GDPR, CCPA, etc.)

    El rol del desarrollador, enfoque "Shift Left"¶

    Tradicionalmente, la seguridad se consideraba al final del proceso de desarrollo. El enfoque "Shift Left" propone integrar la seguridad desde el inicio del desarrollo:

    • Prevención temprana: Corregir problemas antes de que se conviertan en vulnerabilidades
    • Reducción de costos: Es más económico identificar problemas en etapas tempranas
    • Mejor calidad de código: La seguridad se convierte en parte integral del código

    El triángulo CIA (Confidencialidad, Integridad, Disponibilidad)¶

    Principios fundamentales de la ciberseguridad:

    1. Confidencialidad (Confidentiality)

- Control de acceso adecuado: usuarios y contraseñas, autenticación multifactor (MFA), permisos basados en roles.
- Cifrado de datos: si alguien roba un archivo cifrado, no podrá leerlo sin la clave.
- Acceso solo para personal autorizado: olo el personal autorizado puede ver cierta información (ej. médicos accediendo a historiales clínicos).

2. Integridad (Integrity)

- Mecanismos para garantizar la exactitud e integridad de los datos: firmas digitales, hashes (ej. SHA-256) para comprobar que un archivo no fue modificado.
- Sistemas para detectar cambios no autorizados: auditorías, logs, sistemas de detección de intrusos.
- Seguimiento de transacciones: mantener un registro de todas las operaciones (ej. blockchain, bases de datos con logs de cambios).

3. Disponibilidad (Availability)

- Continuidad del servicio: respaldos, planes de recuperación ante desastres.
- Capacidad para manejar ataques y fallos: protección contra ataques DDoS, redundancia en servidores.
- Rendimiento adecuado: mantenimiento, monitoreo y escalabilidad para evitar caídas.

    Amenaza, vulnerabilidad, riesgo¶

    Es fundamental comprender las ciberamenazas, vulnerabilidades y riesgos:

    • Amenaza (Threat): evento o actor que puede causar daño a un sistema.

    • Vulnerabilidad (Vulnerability): punto débil en un sistema que puede ser explotado por un atacante.

    • Riesgo (Risk): probabilidad e impacto de que una amenaza explote una vulnerabilidad para causar daño.

    Diseño seguro¶

    Seguridad desde el Diseño (Security by Design)¶

    La seguridad no es un añadido, sino que debe ser parte integral del diseño. Para ello, siempre se deben aplicar los siguientes puntos:

    • Integrar la seguridad desde el inicio del desarrollo
    • Aplicar los principios básicos de seguridad
    • Utilizar configuraciones seguras por defecto

    Principio de Mínimo Privilegio¶

    Los usuarios y procesos solo deben tener los permisos mínimos necesarios para realizar sus tareas:

    • Separación de cuentas de usuario
    • Limitación de privilegios administrativos
    • Restricción de permisos de procesos

    1. Control de Acceso
    2. Autenticación robusta de usuarios
    3. Permisos y roles de usuario

    4. Gestión segura de sesiones

    7. Procedimientos de eliminación de datos

    10. Validación de entradas (tanto en cliente como en servidor)
    11. Codificación de salida

    12. Consultas preparadas (prepared statements)

    Herramientas básicas¶

    • OWASP ZAP: proxy de interceptación que te permite detectar vulnerabilidades de seguridad en aplicaciones web.
    • Burp Suite Community: herramienta de interceptación de tráfico web que te permite detectar vulnerabilidades de seguridad en aplicaciones web.
    • Nmap: herramienta de escaneo de red que te permite detectar vulnerabilidades de seguridad en aplicaciones web.
    • Wireshark: herramienta de análisis de tráfico que te permite detectar vulnerabilidades de seguridad en aplicaciones web.
    • GitHub Security: herramienta de análisis de código que te permite detectar vulnerabilidades de seguridad en aplicaciones web.
