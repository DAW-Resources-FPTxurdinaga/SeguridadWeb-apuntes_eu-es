    2.2 Seguridad de autenticación y sesiones¶

    Gestión segura de contraseñas¶

    ¿Por qué no es suficiente almacenar contraseñas directamente?¶

    • Si los datos son robados, las contraseñas de los usuarios estarían disponibles
    • Los usuarios reutilizan contraseñas en múltiples servicios
    • Incluso los usuarios legítimos no podrían ver las contraseñas

    Cifrado de contraseñas¶

    Usando bcrypt (Python)¶

    import bcrypt

def crear_hash_contraseña(contraseña: str) -> bytes:
    # Generar sal aleatoria
    sal = bcrypt.gensalt()
    # Generar hash de la contraseña
    hash_resultado = bcrypt.hashpw(contraseña.encode('utf-8'), sal)
    return hash_resultado

def verificar_contraseña(contraseña: str, hash_almacenado: bytes) -> bool:
    # Verificar contraseña
    return bcrypt.checkpw(contraseña.encode('utf-8'), hash_almacenado)

# Ejemplo
contraseña_usuario = "mi_contraseña_secreta"
hash_almacenado = crear_hash_contraseña(contraseña_usuario)
print(f"Hash almacenado: {hash_almacenado.decode()}")

# Verificar
coincide = verificar_contraseña("contraseña_incorrecta", hash_almacenado)
print(f"Contraseña incorrecta: {coincide}")  # False

coincide = verificar_contraseña(contraseña_usuario, hash_almacenado)
print(f"Contraseña correcta: {coincide}")  # True

    ¿Por qué es importante la sal (Salt)?¶

    • Genera hashes diferentes para contraseñas idénticas
    • Previene ataques con tablas arcoíris (rainbow tables)
    • Los atacantes no pueden identificar usuarios con el mismo hash

    Gestión segura de sesiones¶

    Configurar cookies seguras¶

    Ejemplo en Express.js:

    const session = require('express-session');
const crypto = require('crypto');

// Generar una clave secreta aleatoria para firmar las sesiones
const SESSION_SECRET = crypto.randomBytes(32).toString('hex');

app.use(session({
    secret: SESSION_SECRET,
    name: 'sessionId',
    resave: false,
    saveUninitialized: false,
    cookie: {
        secure: true,      // Enviar solo a través de HTTPS
        httpOnly: true,    // No accesible desde JavaScript
        sameSite: 'lax',   // Protección contra CSRF
        maxAge: 1000 * 60 * 60 * 24, // 1 día
        domain: '.tudominio.com'
    }
}));

    JWT (JSON Web Tokens)¶

    Crear y verificar un token (Node.js):

    const jwt = require('jsonwebtoken');
const crypto = require('crypto');

// Cargar desde variables de entorno o generar una clave secreta fuerte
const JWT_SECRET = process.env.JWT_SECRET || crypto.randomBytes(64).toString('hex');

// Crear un token
function crearToken(usuario) {
    return jwt.sign(
        {
            sub: usuario.id,
            email: usuario.email,
            // No incluir información sensible
        },
        JWT_SECRET,
        { 
            expiresIn: '1h',
            issuer: 'TuAplicacion',
            audience: 'tuaplicacion.com'
        }
    );
}

// Verificar un token
function verificarToken(token) {
    try {
        const decoded = jwt.verify(token, JWT_SECRET, {
            issuer: 'TuAplicacion',
            audience: 'tuaplicacion.com'
        });
        return { valido: true, contenido: decoded };
    } catch (error) {
        return { 
            valido: false, 
            error: error.message 
        };
    }
}

// Ejemplo
const usuario = { id: 123, email: 'usuario@ejemplo.com' };
const token = crearToken(usuario);
console.log('Token generado:', token);

const resultado = verificarToken(token);
console.log('Token verificado:', resultado);

    Autenticación Multifactor (MFA)¶

    ¿Por qué usar MFA?¶

    • Previene accesos no autorizados incluso si se compromete la contraseña
    • Combina "algo que sabes" + "algo que tienes" + "algo que eres"

    Tipos de MFA:¶

    1. Aplicaciones autenticadoras (Google Authenticator, Authy, Microsoft Authenticator)
    2. Código SMS (menos seguro debido a ataques de SIM swapping)
    3. Llaves de seguridad físicas (YubiKey, Titan Security Key)
    4. Aplicaciones requeridas (apps bancarias, Google Prompt)

    Implementación de OTP (Contraseña de un solo uso)¶

    import pyotp
import qrcode

def configurarMFA(nombre_usuario):
    # Generar una nueva clave secreta para el usuario
    clave = pyotp.random_base32()

    # Crear URI para generar código QR
    totp_uri = pyotp.totp.TOTP(clave).provisioning_uri(
        name=nombre_usuario,
        issuer_name='Tu Aplicación'
    )

    # Generar código QR
    qr = qrcode.make(totp_uri)
    qr.save(f'mfa_{nombre_usuario}.png')

    return clave

def verificarCodigo(clave, codigo_usuario):
    totp = pyotp.TOTP(clave)
    return totp.verify(codigo_usuario)

# Ejemplo de uso
usuario = "persona@ejemplo.com"
clave = configurarMFA(usuario)
print(f"Escanee el código QR con Google Authenticator: mfa_{usuario}.png")

# Simular ingreso de código por el usuario
codigo_usuario = input("Ingrese el código de su aplicación de autenticación: ")
if verificarCodigo(clave, codigo_usuario):
    print("¡Autenticación exitosa!")
else:
    print("Código incorrecto, intente nuevamente.")

    Consejos de seguridad para sesiones¶

    1. Usar identificadores de sesión largos y aleatorios
    2. Establecer tiempos de expiración apropiados
    3. Permitir solo una sesión activa por usuario
    4. Implementar cierre de sesión obligatorio
    5. Monitorear la ubicación y navegador habituales del usuario
    6. Mostrar al usuario sus sesiones activas
